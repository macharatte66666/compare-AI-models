🔥 1. システムプロンプトの強化 HIGH IMPACT +3pt

現在のコードでは systemContent は空文字または最小限。 詳細なシステムプロンプトを注入するだけで解答精度・分析力が大きく向上します。

具体的な追加内容：
① タスク分類（コード / 分析 / 一般 / 数学）を明示
② 「段階的に考えてから答えよ（Chain-of-Thought）」の指示
③ 出力フォーマット指定（Markdown, JSON等）
④ 日本語優先の言語指示
// バックエンド chat.js のsystemContentを拡張 const baseSystem = `You are a highly capable AI assistant. Think step-by-step before answering. For code: write clean, commented, production-ready code. For analysis: structure your reasoning clearly. Respond in the same language as the user.`; let systemContent = body.systemContent || baseSystem; // フォールバックとして注入

⚡ 3. Chain-of-Thought プロンプト注入 MED IMPACT +2pt

複雑な質問に対して「考える手順」をプロンプトに埋め込む。 これはモデル変更なしで複雑問題対応力を最も効率よく上げる方法です。

実装方法：
① 複雑クエリ（長文・多段階）を検出
② ユーザーメッセージの前に CoT指示を追加
③ 「まず問題を分解し、各ステップを明示してから最終回答を出せ」
const isComplex = lastText.length > 120 || /なぜ|理由|分析|比較|evaluate|explain|analyze/i.test(lastText); if (isComplex) { chatMessages = [ ...chatMessages.slice(0, -1), { role: 'user', content: `問題を段階的に分解して考えてください。\n\n` + lastUserMsg.content } ]; }

⚡ 4. 会話履歴の最適化（トークン効率） MED IMPACT +1〜2pt

長い会話では古い履歴がコンテキストを圧迫し、後半の回答精度が低下します。 スライディングウィンドウで直近N件のみ送信することで精度が安定します。

実装方針：
① 直近8〜12ターンのみ保持
② Systemプロンプト + 最新ユーザーメッセージは常に保持
③ 古いメッセージは要約してからコンテキストに残す（発展版）
// トークン節約: 直近10件に制限 const MAX_HISTORY = 10; if (chatMessages.length > MAX_HISTORY) { chatMessages = chatMessages.slice( chatMessages.length - MAX_HISTORY ); }

💡 6. エラー時の再試行（Retry）ロジック LOW +1pt

現在のフォールバックは「Groq失敗→Gemini切替」の1段階のみ。 指数バックオフ付きリトライを追加することで、 一時的なAPIエラーによる応答品質の低下を防げます。

実装方針：
① 同モデルで最大2回リトライ（500ms, 1000ms待機）
② 2回失敗後にフォールバック発動
③ Vercelのタイムアウト（300秒）内に収まるよう設計
async function withRetry(fn, retries=2) { for (let i=0; i<=retries; i++) { try { return await fn(); } catch(e) { if (i===retries) throw e; await new Promise(r=>setTimeout(r, 500*(2**i))); } } }
